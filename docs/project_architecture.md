# Project Architecture

This project is organized as a reproducible modeling pipeline for problem B.

## Environment

Use the local Conda environment `pytorch_gpu`.

```powershell
conda activate pytorch_gpu
```

## Directory Layout

```text
datasets/                 Raw CSV data files.
src/                      Reusable Python modules.
  dataset_config.py       Dataset input/output definitions and loaders.
  problem1_analysis.py    DP-FSM feature analysis.
  problem2_modeling.py    MCAS-PM model training and evaluation.
  problem3_explainability.py
                           TPCI-EM explainability and physical consistency.
scripts/                  Command-line workflow entry points.
  run_problem1_analysis.py
  run_problem2_modeling.py
  run_problem3_explainability.py
  run_all.py
reports/                  Generated tables, figures, and Markdown summaries.
```

## Workflow Order

1. `python scripts/run_problem1_analysis.py`
2. `python scripts/run_problem2_modeling.py`
3. `python scripts/run_problem3_explainability.py`

Or run the complete pipeline:

```powershell
python scripts/run_all.py
```

## Design Notes

- `dataset_config.py` is the single source of truth for dataset mappings.
- Problem 2 reuses the selected features generated by problem 1.
- Problem 3 reuses the full-feature best models selected by problem 2.
- Reports are committed so the paper can cite stable tables and figures.
